# fit Docs Research

- Verify version-sensitive behavior before code depends on it.
- Use Context7 first for third-party Android and Kotlin ecosystem docs.
- Use `openaiDeveloperDocs` only for Codex, OpenAI platform, skills, AGENTS, MCP, and multi-agent questions.
- Prefer exact API names, signatures, config keys, and migration notes over loose summaries.
- If docs disagree or are unclear, surface the uncertainty explicitly.
- Keep answers short and implementation-oriented.
